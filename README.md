# PPI Scout

PPI Scout is a local workflow for planning, executing, and reviewing
protein-complex and motif-peptide structure predictions with Boltz. It combines
a deterministic command-line interface, an optional Codex Skill, and an
offline result viewer.

PPI Scout records resolved inputs, representation decisions, MSA policy,
execution parameters, controls, logs, and reports. It does not treat model
confidence as experimental evidence of binding.

## Components

| Component | Role | Required |
|---|---|---|
| PPI Scout CLI | Input validation, routing, peptide design, job compilation, execution records, and reporting | Yes |
| Boltz | Structure prediction backend | Only for live prediction |
| PPI Scout Codex Skill | Guided planning and interpretation in Codex | Optional |
| Offline HTML report | Local review of inputs, run state, candidates, controls, and confidence fields | Generated by PPI Scout |

## Capabilities

- Resolve protein names, sequences, and single-record FASTA inputs into an
  auditable job definition.
- Route a question to full-length, domain, motif-peptide, or manual-review
  workflows without using protein length as the sole criterion.
- Scan a resolved sequence for canonical AIM/LIR candidates and rank them with
  transparent sequence-only features.
- Generate nested peptide windows with anchor-mutant, reverse-core, and
  composition-matched controls.
- Compile Boltz YAML and command arguments without silently changing MSA
  policy.
- Execute dry runs by default and require an explicit `--live` flag for model
  inference.
- Expand a reviewed motif hypothesis into a complete matched-control Boltz
  panel, stream progress, resume unfinished tasks, and write all summaries
  with one `run-panel` command.
- Collect confidence fields and generate Markdown and self-contained HTML
  reports.
- Provide Chinese, English, and Japanese interactive prompts.

## Requirements

| Use case | Requirement |
|---|---|
| Planning, motif scanning, peptide design, and visualization | Python 3.10 or later |
| Live Boltz prediction | Python 3.10-3.12 and a compatible Boltz installation |
| Practical local inference | A supported accelerator; NVIDIA CUDA and Apple Silicon MPS are supported paths |
| Windows GPU execution | WSL2 with Ubuntu is recommended |

Boltz supports CPU execution, but the upstream project notes that it is
significantly slower than GPU execution.

## Installation

Install the released CLI directly from GitHub:

```bash
python -m pip install "git+https://github.com/ZHULUYI-KyushuU/ppi-scout.git@v0.4.0"
```

For repository-scoped Codex Skill support, clone or download the complete
source repository instead. Then create an isolated environment from the
repository root:

```bash
git clone https://github.com/ZHULUYI-KyushuU/ppi-scout.git
cd ppi-scout
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install .
```

PowerShell users can run the same installation without activating the
environment:

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install .
```

Install Boltz in the same environment when live prediction is required.

CUDA installation:

```bash
python -m pip install --upgrade "boltz[cuda]"
```

CPU-only installation:

```bash
python -m pip install --upgrade boltz
```

Boltz downloads model data on first use. Its default cache is `~/.boltz`; set
`BOLTZ_CACHE` to an absolute path to use another location. Refer to the
[official Boltz repository](https://github.com/jwohlwend/boltz) and
[prediction documentation](https://github.com/jwohlwend/boltz/blob/main/docs/prediction.md)
for current upstream requirements.

Validate the installation:

```bash
ppi-scout --lang en doctor
boltz predict --help
```

`ready_to_plan: true` confirms that the PPI Scout planning stack is available.
`ready_to_run: true` additionally confirms that the Boltz executable is
discoverable.

## Codex integration

Open the repository root in Codex and select PPI Scout from the available
Skills. Explicit invocation is `@ppi-scout`; clients that expose Skills as
prompt variables may display `$ppi-scout`. The Skill provides workflow
guidance; local planning, scanning, execution, and reporting are performed by
the installed PPI Scout CLI and Boltz backend.

Do not submit confidential sequences to an external AI service. Run the CLI
directly in a local terminal when project policy requires inputs to remain on
the workstation.

## Core workflow

Create a job from exact FASTA inputs:

```bash
ppi-scout --lang en plan \
  --fasta-a receptor.fasta \
  --fasta-b partner.fasta \
  --organism "Organism and strain" \
  --mode auto \
  --name receptor-partner \
  -o job.json
```

Compile and review the job without starting inference:

```bash
ppi-scout run job.json \
  --output-dir runs/receptor-partner \
  --dry-run
```

Start the reviewed prediction explicitly:

```bash
ppi-scout run job.json \
  --output-dir runs/receptor-partner \
  --live
```

Collect confidence fields and generate a report:

```bash
ppi-scout analyze runs/receptor-partner
ppi-scout --lang en report runs/receptor-partner
```

`run` and `resume` attempt to generate `report.html` in the run directory.
The file is self-contained and can be opened directly in a browser. Use
`visualize` to regenerate it or to render a scan/job JSON file:

```bash
ppi-scout --lang en visualize runs/receptor-partner
```

## One-command motif panel execution

After `plan` has produced and you have reviewed a `motif_peptide` job, compile
the complete WT/control panel without running Boltz:

```bash
ppi-scout run-panel job.json \
  --windows 24 \
  --output-dir runs/receptor-motif-panel \
  --dry-run
```

Start every unfinished panel task with one explicit command:

```bash
ppi-scout run-panel job.json \
  --windows 24 \
  --output-dir runs/receptor-motif-panel \
  --live
```

`run-panel` generates independent WT, anchor-mutant, `AAAA`, flank-scramble,
composition-matched scramble, and reverse-decoy tasks under identical backend
settings. It streams Boltz output, skips tasks already marked complete when the
same command is rerun, and automatically writes `panel.json`, `plan.json`,
`confidence_summary.csv`, `report.md`, and `report.html`.

The default is local single-sequence mode (`msa: empty`) and does not enable a
remote MSA service. Supply `--receptor-msa /path/to/receptor.a3m` for an audited
local MSA. `--remote-msa` is the explicit sequence-upload authorization.
On Apple Silicon, `--accelerator auto` selects the MPS-backed GPU path and
disables unsupported specialized kernels.

## AIM/LIR workflow

Candidate scanning is opt-in and requires a resolved amino-acid sequence:

```bash
ppi-scout scan-motifs --fasta partner.fasta -o motif-scan.json
```

Review the complete candidate list before selecting any candidate. Candidate
IDs follow sequence position; the `rank` field is a separate sequence-only
priority. Generate a peptide panel only after an explicit selection:

```bash
ppi-scout scan-motifs \
  --fasta partner.fasta \
  --design-candidate CANDIDATE_ID \
  --windows 16,24,34 \
  --seed 7 \
  -o peptide-panel.json
```

The canonical sequence scan cannot establish motif accessibility, disorder,
topology, biological function, or binding.

## Public example

The bundled example uses the established *Saccharomyces cerevisiae*
Atg8–Atg19 system with reviewed UniProtKB sequences (`P38182` and `P35193`).
It includes a resolved job, a complete AIM scan, an evidence-selected peptide
panel, an exploratory one-command Atg8–Yta7 FDFL panel, and commands for
generating the offline HTML view. See
[Example Workflows](examples/README.md).

## MSA policy

PPI Scout does not enable a remote MSA service by default.

- Use a precomputed local MSA when provenance and confidentiality must remain
  under local control.
- Use `msa: empty` for single-sequence inference and record the expected
  accuracy tradeoff.
- Add `--remote-msa` only after authorizing external sequence transmission.

Never store MSA credentials, private sequences, model caches, or run outputs in
the repository.

## Interpretation boundary

PPI Scout reports structural-model evidence, not experimental binding.

- Do not interpret ipTM, protein ipTM, pLDDT, confidence score, or rank score as
  proof of interaction.
- Review the predicted interface and compare WT against matched negative
  controls under identical settings.
- Do not use the Boltz affinity module for protein-protein or protein-peptide
  affinity. The upstream affinity model is defined for small-molecule ligands
  against protein targets.
- Confirm model-derived hypotheses with independent structural, biochemical,
  genetic, or cell-biological evidence.
